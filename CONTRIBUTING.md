# Contributing

Contributions should preserve the documentation standards defined in docs/reference/DOCUMENTATION_STANDARD.md and the shared terminology defined in docs/reference/TERMINOLOGY.md.

## Documentation Types

Contributions should fit one existing documentation type: policy, engineering guidance, operational checklist, reference, or example.

## Documentation Review

Documentation changes should be reviewed for security accuracy, engineering practicality, maintainability, and developer usability.

## Single Responsibility

Each document should keep one clear responsibility. New content should be added to the document that owns the topic.

## Avoid Duplication

Prefer cross-references over repeated content. Do not duplicate guidance that already belongs to another document.
